# Content Architecture
## Release 0.1 — Landing Experience

Version: 1.1
Status: Approved

# Purpose

This document defines the storytelling, content hierarchy and persuasion strategy for the first public version of the Felipe Santander Platform.

The objective is not to present a résumé.
The objective is to progressively build confidence until the visitor naturally wants to start a conversation.

# Primary Audiences

1. Recruiter / HR
2. Head of Product
3. CTO / CPO
4. CEO

# Core Communication Principle

Always begin with the visitor's problem.

Never begin with Felipe.

# Narrative Structure

Problem
↓
Recognition
↓
Approach
↓
Evidence
↓
Trust
↓
Urgency
↓
Conversation

# Hero

Create curiosity.

# Problem

Products rarely fail because of lack of talent.
They fail because Product, Business and Engineering stop sharing the same vision.

# Approach

Present four pillars:
- Strategy
- Design
- Technology
- Artificial Intelligence

# Transformations

Each case should answer:
- Challenge
- Contribution
- Impact

# Testimonials

Reinforce:
- Technical credibility
- Leadership
- Collaboration

# About

Write in first person.
Explain philosophy, not chronology.

# CTA

Invite a conversation.

# Emotional Journey

Curiosity → Recognition → Confidence → Admiration → Urgency → Action

# Success Metrics

- Reach Transformations
- Click LinkedIn
- Download CV
- Contact

# Final Thought

'I expected a portfolio. I found someone capable of helping transform how our organization builds digital products.'
